# 俄语百词斩开发文档

本文档是当前工程入口版开发说明。完整产品范围、数据模型和验收标准以根目录 `俄语版_智能词汇学习产品需求与开发文档.md` 为准，开发约定以根目录 `AGENTS.md` 为准。

## 开发步骤

开发最多拆成 8 步：

1. 工程基线
2. 俄语词库与导入
3. 账户与新手引导
4. 首次学习闭环
5. 复习与错词闭环
6. 排行榜与打卡
7. PWA 与离线同步
8. 上线验收

每完成一步必须汇报已完成内容、下一步和总体百分比。

## 第 1 步工程基线验收

第 1 步完成时应具备：

- pnpm workspace；
- React + TypeScript + Vite Web/PWA 应用壳；
- Fastify API health check；
- `packages/contracts` 和 `packages/domain` 共享包；
- ESLint、Prettier、TypeScript、Vitest；
- PostgreSQL 与 Redis 本地 Compose 入口；
- Prisma schema 校验入口；
- `pnpm format:check`、`pnpm lint`、`pnpm typecheck`、`pnpm test`、`pnpm build` 可运行。

## 第 2 步词库与导入验收

第 2 步完成时应具备：

- 人教版俄语词库字段：出版社、教材版本、册别、单元编号、单元名称、课次、单词；
- `packages/contracts` 中的词库与俄语词条 Schema；
- `packages/domain` 中的人教版示例词库、单元分组和导入行校验；
- `scripts/import-vocabulary` 的 CSV dry-run 校验入口；
- API 能返回词库列表和册别单元；
- Web 能展示人教版俄语词库与单元；
- Prisma schema 包含词库、单元和俄语词条模型。

## 第 3 步账户与新手引导验收

第 3 步完成时应具备：

- 访客学习者模型与共享合约；
- 用户偏好包含学段、年级、册别、单元、每日新词目标和提醒开关；
- 领域层能根据人教版册别与单元生成活跃学习计划；
- API 能创建访客、创建学习计划、查询当前活跃学习计划；
- Web 能完成“访客开始学习 -> 生成学习计划”的新手引导；
- Prisma schema 包含学习者、偏好和学习计划模型。

## 第 4 步首次学习闭环验收

第 4 步完成时应具备：

- 老师演示账号与登录学习者演示账号；
- 领域层能从学习计划生成俄语词卡和主动回忆题；
- 学习会话完成后生成已背词数、掌握词数和正确率；
- API 能创建学习会话、完成学习会话、查询学习结果；
- 老师进度接口能同时查看登录学习者和游客的背诵进度；
- Web 能完成“生成计划 -> 完成首组背诵 -> 老师账号查看进度”的闭环；
- Prisma schema 包含老师账号、学习会话、作答记录和单词进度模型。

## 第 5 步复习与错词闭环验收

第 5 步完成时应具备：

- 领域层实现 SRS v1 四种作答等级的间隔计算；
- 作答能更新单词掌握状态、连续正确、正确/错误次数和下次复习时间；
- 答错后自动进入错词本，并按错词优先生成到期复习队列；
- 错词消灭模式连续 3 次正确后移出错词本；
- API 能返回复习队列、错词本，并支持错词消灭训练；
- Web 能展示错词进入、10 分钟后复习和错词消灭结果；
- Prisma schema 单词进度包含 SRS 参数、错误类型和平均响应时间。

## 第 6 步排行榜与打卡/轻量教师任务验收

第 6 步完成时应具备：

- 领域层能从服务端确认的学习结果生成有效背词积分事件；
- 打卡必须基于已完成的当天学习结果，能计算连续打卡天数；
- 排行榜支持今日榜、周榜、册别榜，并保留班级榜聚合能力；
- 学习看板能展示今日已背、今日掌握、待复习、错词、今日积分、本周积分和近 7 天趋势；
- 老师账号能把登录学习者和游客加入自己的学生列表；
- 老师能按人教版册别与单元布置背词任务，任务记录每日新词目标、截止日期和学生范围；
- 老师能针对任务给学生发布评价，并在任务概览中看到评价内容；
- 任务概览能结合学习结果展示每个学生的已背词数、掌握词数、完成率和正确率；
- API 提供打卡、排行榜、学习看板、添加学生、布置任务、发布评价和任务概览接口；
- Web 能完成“完成首组背诵 -> 完成今日打卡 -> 查看学习看板和排行榜”的学习激励闭环；
- Web 老师看板能完成“查看进度 -> 添加学生 -> 布置背词任务 -> 评价学生”的班级任务闭环；
- Prisma schema 包含学习积分事件、打卡记录、老师学生、任务、任务分配和教师评价模型。

## 第 7 步 PWA 与离线同步验收

第 7 步完成时应具备：

- PWA 通过 vite-plugin-pwa 提供应用安装入口和静态资源预缓存；
- Workbox 只对人教版词库接口使用 NetworkFirst 缓存，不缓存打卡、排行榜等服务端确认结果；
- 领域层能生成离线学习包，离线包包含学习会话、词卡、主动回忆题、缓存时间和过期时间；
- 领域层能生成离线同步操作，记录 idempotency key、创建时间、retryCount、lastError 和队列状态；
- 离线同步队列按创建时间提交，并按 idempotency key 去重；
- API 提供 `/api/v1/offline/sync`，恢复网络后复用学习完成逻辑，重复同步不会重复计分；
- Web 能完成“缓存离线学习包 -> 模拟断网作答 -> 恢复网络同步”的闭环；
- 离线作答只进入待同步队列，不直接更新最终打卡和排行榜；
- 服务端确认同步后才更新学习结果，并允许用户继续完成打卡和榜单更新；
- Prisma schema 包含离线同步操作模型，保留未来服务端持久化队列能力。

## 第 8 步上线验收

第 8 步完成时应具备：

- API 响应统一包含安全响应头，降低 MIME 嗅探、点击劫持和权限误用风险；
- 写请求具备统一限流能力，超限返回 `RATE_LIMIT_EXCEEDED` 和 `Retry-After`；
- 领域层提供敏感日志字段脱敏工具，覆盖 token、Cookie、邮箱、手机号等字段；
- 领域层提供 MVP 上线验收检查清单，覆盖安全、隐私、性能、可访问性、内容、备份和 e2e；
- API 提供 `/api/v1/ops/readiness`，用于返回当前上线验收报告；
- Web 提供隐私政策、用户协议入口，以及清除本机学习数据的账号控制；
- 根目录提供 `pnpm release:check`，串联格式、lint、类型、单元测试、构建、Prisma、词库校验和 e2e；
- `docs/RELEASE_CHECKLIST.md` 记录发布前配置、命令、安全、内容、PWA、备份恢复和上线条件；
- README 同步上线验收命令；
- `.env.example` 提供限流配置样例；
- 所有第 1-8 步关键验证命令通过。
