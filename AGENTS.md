# 俄语版百词斩开发约定

## 沟通约定

- 与用户交流、进度汇报、技术说明和文档默认使用中文。
- 代码标识符、接口字段、提交中的技术名词使用英文。
- 涉及俄语内容时，俄语原文保留西里尔字母，中文说明用于解释含义和学习规则。
- 遇到需求、内容来源、未成年人保护或工程约束冲突时，直接说明冲突、影响和推荐方案，不默默猜测。

## 项目简介

本项目是面向中国初中、高中俄语学习者的百词斩式智能词汇学习产品。首期形态为 **React + TypeScript 响应式 Web/PWA**。

核心闭环：

```text
选择学段与册别 → 生成计划 → 学习新词 → 主动回忆
→ SRS 复习 → 错词强化 → 打卡 → 排行榜激励
```

产品与开发依据见：

- [`俄语版_智能词汇学习产品需求与开发文档.md`](俄语版_智能词汇学习产品需求与开发文档.md)

## 事实来源优先级

出现不一致时，按以下顺序执行：

1. 用户在当前任务中的明确要求；
2. `俄语版_智能词汇学习产品需求与开发文档.md`；
3. 已有代码、测试和接口约定；
4. 本文件中的通用开发约定。

不要自行把后续演进功能提升为 MVP 必做项。用户特别要求“开发不要分很多步骤，最多 10 步完成”，所以所有开发计划、任务拆解和阶段汇报都必须控制在 10 步以内。

## 技术基线

- pnpm workspace
- React + TypeScript + Vite
- React Router
- TanStack Query
- Zustand
- React Hook Form + Zod
- Tailwind CSS
- vite-plugin-pwa + Workbox
- IndexedDB + Dexie
- Node.js + TypeScript + Fastify
- Prisma + PostgreSQL
- Redis
- Vitest + Testing Library + Playwright
- ESLint + Prettier

创建项目时使用稳定版本并锁定依赖。不要擅自替换核心框架或并行引入同类库。

## 预期项目结构

```text
apps/
├── web/                  # React Web/PWA
└── api/                  # Fastify REST API
packages/
├── contracts/            # 前后端共享 DTO 与 Zod Schema
├── domain/               # SRS、掌握度、计划、排行榜积分等纯领域逻辑
└── config/               # 共享 TypeScript/ESLint 配置
scripts/
└── import-vocabulary/    # 俄语词库导入与校验
docs/
└── DEVELOPMENT.md        # 如后续拆分开发文档，可放在这里
```

按业务能力组织代码，避免把所有组件、接口或工具函数堆入单一大文件。

## 开发路线

开发最多拆成以下 8 步，不再细分成大量阶段：

1. **工程基线**：建立 workspace、Web/API、共享包、代码规范、测试框架和健康检查。
2. **俄语词库与导入**：建立初中俄语、高中俄语按册模型，完成 CSV 导入、字段校验和来源记录。
3. **账户与新手引导**：完成登录/访客体验、学段年级册别选择、每日词量和目标日期设置。
4. **首次学习闭环**：完成今日任务、俄语词卡、发音/重音/语法提示、基础主动回忆题和学习结果。
5. **复习与错词闭环**：实现 SRS v1、到期复习、错词本、消灭模式、幂等学习记录和会话恢复。
6. **排行榜与打卡**：实现有效背词积分、今日榜、周榜、册别榜、打卡日历和学习看板。
7. **PWA 与离线同步**：实现安装、学习包缓存、离线作答队列、恢复网络后的幂等同步。
8. **上线验收**：完成安全、限流、日志脱敏、性能、无障碍、内容校验、备份恢复和端到端验收。

每完成一步，汇报：

- 已完成什么；
- 下一步是什么；
- 总体完成百分比。

## MVP 范围

必须实现：

- 初中俄语、高中俄语按人教版教材册别与单元组织的词库；
- 词库列表、册别详情、学习计划；
- 老师添加学生、按人教版单元布置背词任务、查看任务进度并评价学生的轻量教师能力；
- 俄语词卡：原形、重音、发音、词性、中文释义、例句和语法提示；
- 主动回忆题：看俄选中、看中选俄、听音选词、俄语拼写、重音位置、名词性别和动词体；
- SRS v1 复习调度；
- 错词本和消灭模式；
- 打卡日历；
- 背单词排行榜；
- 学习看板；
- PWA 基础离线能力；
- 词库导入校验和内容来源记录。

暂不实现：

- 完整机构后台、排课收费和多班级组织管理；
- 公开陌生人社交；
- 付费会员和广告权益；
- AI 造句批改、AI 对话、发音评分；
- 用户上传自定义词库；
- 完整 FSRS 参数训练；
- 多语言扩展。

## 俄语内容约定

- 词汇按“人教版教材 → 学段 → 年级/层级 → 册别 → 单元 → 课次 → 单词”组织。
- 单元分类以授权的人教版俄语教材词表为准，不凭记忆手写单元名称或课文内容。
- 导入数据必须保留人教版教材版本、册别、单元编号、单元名称和课次字段。
- 每个单词至少包含 `lemma`、`stressedLemma`、`partOfSpeech`、`definitionZh`、`bookSlug`、`unit`、`lesson`。
- 例句、音频、图片和教材词表必须有可追踪来源。
- 拼写题默认忽略重音符号，重音题单独校验重音位置。
- 名词优先记录性、复数和常见格变化提示。
- 动词优先记录完成体/未完成体、体对和变位类型。
- 不直接修改生产数据库中的词库内容，必须通过导入脚本验证后导入。

## 排行榜约定

- 排行榜只统计服务端确认后的有效学习行为。
- 积分依据“真实掌握”和“稳定坚持”，不按单纯点击次数排名。
- 同一单词当天重复刷正确，只计一次主要积分。
- 未到期复习的重复作答不进入核心积分。
- 排行榜必须有每日积分上限和异常刷榜处理。
- 初中用户默认不开启陌生人全站榜，只开放同册匿名榜或班级榜。
- 用户可隐藏自己在公开榜单中的展示。
- 排行榜不得展示手机号、邮箱、学校详细信息等敏感数据。

推荐积分公式：

```text
有效背词积分 =
  新增掌握词数 × 10
  + 到期复习完成词数 × 3
  + 错词消灭词数 × 8
  + 连续打卡加成
  - 无效重复答题惩罚
```

## 领域逻辑约定

以下逻辑放在 `packages/domain`，写成纯函数并优先测试：

- SRS 间隔计算；
- 掌握状态转换；
- 今日任务排序；
- 学习计划日期计算；
- 俄语拼写与重音判定；
- 错词消灭规则；
- 连续打卡计算；
- 排行榜积分计算；
- 离线操作去重。

React 组件不得自行计算另一套复习规则或排行榜积分。客户端展示服务端返回的最终结果。

## 服务端事实来源

- 学习进度、打卡、排行榜积分和学习计划以服务端数据为准。
- IndexedDB 只保存缓存、未完成会话和待同步操作。
- 本地完成但未同步时必须明确显示“待同步”。
- 写请求使用幂等键，重试不能重复累计学习次数或排行榜积分。
- 学习记录写入、进度更新和积分事件生成应保持事务一致。

## TypeScript 约定

- 启用 `strict`。
- 核心业务代码禁止使用 `any`，外部未知数据先用 `unknown` 并校验。
- 类型优先从 Zod Schema 推导，避免手写两份不一致的 DTO。
- 使用字符串联合类型表达有限状态。
- 数据库时间统一使用 UTC ISO 8601 字符串传输。
- 计数、时长和积分字段在名称中体现单位或含义，如 `responseMs`、`scoreDelta`、`intervalDays`。
- 不使用 TypeScript `enum` 承载跨接口协议值，使用 `as const` 或字符串联合。

## React 约定

- 页面放在对应 `features` 模块中，路由层只负责组合和守卫。
- 服务器状态使用 TanStack Query，不复制进 Zustand。
- Zustand 只保存当前学习会话、离线队列和轻量 UI 状态。
- 表单统一使用 React Hook Form + Zod。
- 所有异步页面必须提供加载、空数据、错误和重试状态。
- 列表项使用稳定业务 ID，不使用数组下标作为可变列表 key。
- 学习页以移动端优先，桌面端保持聚焦列。

## 样式与交互约定

- 以 375px 宽度作为主要移动端检查视口。
- 学习内容桌面端最大宽度建议为 720px。
- 点击区域不小于 44×44px。
- 正误反馈不能只依赖红绿颜色。
- 动画必须尊重 `prefers-reduced-motion`。
- 不使用悬停作为唯一操作方式。
- 俄语拼写题在手机软键盘弹出时仍能看到输入框与提交按钮。
- 排行榜解释字段要清晰展示，避免用户不知道为什么排名变化。

## API 约定

- 路径前缀统一为 `/api/v1`。
- 请求和响应通过 `packages/contracts` 中的 Schema 校验。
- 错误使用稳定业务代码和面向用户的中文消息。
- 不暴露 SQL、堆栈、内部路径或第三方密钥。
- 列表默认使用游标分页。
- 写入学习记录必须携带 `Idempotency-Key`。
- 修改接口时同步更新 contracts、调用方、测试和文档。

标准错误结构：

```json
{
  "error": {
    "code": "VALIDATION_FAILED",
    "message": "提交内容不完整，请检查后重试。",
    "requestId": "req_123"
  }
}
```

## PWA 与离线约定

- Service Worker 更新不能静默破坏进行中的学习会话。
- 当前会话写入 IndexedDB 后才允许显示为可离线继续。
- 待同步记录保留幂等键、创建时间、重试次数和最后错误。
- 鉴权失效时暂停同步，不能无限重试。
- 不把“写入本地队列”展示为“服务端同步成功”。
- 离线时不得更新最终打卡状态或排行榜名次。
- 恢复网络后，服务端确认学习记录后再更新积分和榜单。

## 测试要求

必测领域逻辑：

- SRS 四种回答等级；
- `easeFactor` 上下界；
- 新词、学习中、模糊、错词、掌握和遗忘状态转换；
- 俄语拼写判定、重音题、名词性别题、动词体题；
- 到期复习排序；
- 学习计划完成日期；
- 跨时区打卡；
- 排行榜积分去重、上限和异常处理；
- 离线队列幂等同步。

必测用户路径：

- 新用户选择初中俄语册别并完成首次学习；
- 高中用户完成到期复习并打卡；
- 答错后进入错词本并完成消灭训练；
- 完成学习后看到排行榜积分变化；
- 刷新后恢复进行中的学习会话；
- 断网作答后恢复网络同步；
- 手机视口完成俄语拼写题。

## 性能与安全

- 首页 LCP 目标小于 2.5 秒。
- 学习卡片交互响应目标小于 200ms。
- 非首屏路由懒加载。
- 不一次性加载完整大型词库。
- 验证码、登录、作答提交和排行榜刷新接口必须限流。
- 刷新令牌仅放入 HttpOnly、Secure Cookie。
- 密钥不得进入代码、日志或提交记录。
- 日志不得包含验证码、令牌、完整手机号或邮箱。
- 用户内容和错误信息在渲染前进行安全处理。
- 未成年人账号默认采用更严格的隐私和榜单展示策略。

## 修改文档的时机

以下变化必须同步更新产品/开发文档和本文件：

- MVP 范围变化；
- 开发步骤数量或阶段边界变化；
- 技术栈或目录结构变化；
- 俄语词库字段、题型或内容来源规则变化；
- SRS、掌握度、打卡或排行榜规则变化；
- 离线同步边界变化；
- 性能、安全或验收标准变化。

## 提交与变更范围

- 每次变更聚焦一个明确目标。
- 不在功能提交中混入无关重构和依赖升级。
- 不修改用户未要求且与任务无关的文件。
- 删除废弃实现，不长期保留注释代码。
- 提交前至少运行与变更相关的 lint、类型检查和测试。
- 宣称任务完成前必须报告实际执行过的验证命令和结果。
- 未运行的检查必须明确说明，不能推测其已通过。

## 完成检查

提交任务前确认：

- 功能符合 MVP 范围；
- 开发拆解不超过 10 步；
- 初中俄语、高中俄语按人教版教材册别与单元分类的规则没有被破坏；
- 俄语专项题型和字段有测试覆盖；
- 排行榜积分由服务端确认并可防重复；
- 数据和 API 类型一致；
- 加载、空状态、错误和重试已处理；
- 手机视口可正常操作；
- 必要测试已添加并通过；
- Lint、类型检查和构建未被破坏；
- 没有新增敏感信息、无来源内容或无效入口；
- 相关工程文档已经同步。
